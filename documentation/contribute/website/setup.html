Documentation: Dev Environment Setup

To contribute to the MigrateDB documentation site you will need to set up your development environment so that you run the website locally and submit changes.

For this you will need to set up Git, Make, Docker and an editor.

Prerequisites

  • Git

Editor

The documentation site uses HTML files with Liquid headers for Jekyll. We personally use IntelliJ, which works quite well, but feel free to use any editor that suits you.

Done!

Congratulations! You now have a fully functional dev environment for the MigrateDB documentation site. You can start contributing!
